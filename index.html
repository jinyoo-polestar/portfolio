다양한 디바이스에서도 일관된 사용자 경험을 위해 반응형 디자인을 적용했습니다.

How to make

➊ BEM 방법론 적용

BEM 방법론을 사용해 유지보수와 관리가 편리한 구조로 설계했습니다.
소규모 프로젝트라 BEM을 사용하지 않는게 더 간편할 수도 있었지만,
협업 시 코드 가독성을 높이고 일관성 있는 관리가 중요할 것이라고 생각되어 BEM 방식을 선택했습니다.

[구현 방식]
- 클래스 이름 첫 부분에 섹션명을 명시해, 클래스만 보고도 어디에 위치해있는지 쉽게 알 수 있도록 했습니다.
- 클래스 명이 지나치게 길어지지 않도록, 섹션별로 블록을 명확히 구분해 관리했습니다.
- 주요 요소에 대해 공통된 네이밍 규칙을 적용해 코드 일관성을 유지했습니다. 제목-title, 소제목- subtitle, 버튼-btn, 아이콘-ic

➋ Javascript를 이용한 input label 디자인

넷플릭스의 인풋 라벨은 입력값이 비어있지 않을 때 라벨의 위치와 사이즈가 작아지는 디자인이 적용되어 있었습니다.
기존에는 UI 프레임워크를 이용해 간단히 구현할 수 있었지만,
이번 프로젝트에서는 직접 만들어보았습니다.

[구현 방식]
- input 요소에 focus가 될 때 active 클래스를 추가해 라벨의 위치와 크기 조절
- input 요소가 blur 되면 입력값 확인 후 값이 없으면 active 클래스 삭제 와 같은 형식으로 만들었습니다.

➌ Javascript를 이용한 slider 구현

영화 리스트는 슬라이더 형태로 구성되어 있으며,
오른쪽에 영화가 남아있을 경우 [>] 버튼이 나타나 클릭 시 우측으로 이동
왼쪽에 영화가 있을 경우 [<] 버튼이 나타나 클릭 시 좌측으로 이동

[구현 방식]
- 버튼 모두 기본 display:none 설정
- 좌측 버튼 클릭하면 슬라이더래퍼 길이만큼 좌측으로 이동
- 우측 버튼 클릭하면 슬라이더래퍼 길이만큼 우측으로 이동
(*이때 슬라이더래퍼의 길이보다 스크롤 길이가 더 적을 경우 스크롤 길이만큼 이동)
